# API 说明

## 当前接口
- `GET /health`：基础健康检查，返回 `{"status": "ok"}`。
- `GET /`：根路由示例，返回欢迎信息。

## 规划中的接口
- 解析任务提交：异步/同步邮件解析入口。
- 结果查询：按任务或邮件 ID 获取结构化结果。
- 配置读取：查询当前生效的模板、关键词、分类规则（只读）。

## 设计约定
- 风格：RESTful，路径使用复数资源名，如 `/tasks`、`/results`。
- 响应：统一包含 `data`、`error` 字段（后续在 `app/core` 中统一处理）。
- 校验：Pydantic 模型定义请求/响应，自动生成 OpenAPI（/docs、/openapi.json）。
- 状态码：2xx 成功，4xx 参数/权限问题，5xx 服务器错误。
