# 安全与配置规范

- 环境变量：使用 `.env` 注入，不提交实际值；模板 `.env.example` 需保持更新。
- 端口暴露：仅前端绑定宿主端口；backend/db 保持内网访问，通过反向代理转发。
- 凭证管理：数据库口令、外部 API Key 通过环境变量或部署平台 Secret 传入，禁止写入 Dockerfile/代码。
- 日志：仅输出到 stdout/stderr，避免在容器内生成持久文件；日志中不记录敏感信息。
- 依赖与供应链：镜像尽量使用官方基础镜像；定期更新依赖与基础镜像版本。
- 访问控制：后续增加鉴权时，推荐在反代层或 API 层统一校验，避免在单个路由散布逻辑。
